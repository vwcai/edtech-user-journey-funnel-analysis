# Dashboard Plan

This dashboard plan translates the user journey findings into a monitoring structure that a business or product analytics team could use after implementing tracking improvements.

## Dashboard Goal

The dashboard should help answer one main question:

Are users moving from course exploration to enrollment with less friction after visibility and tracking improvements are implemented?

## Recommended Dashboard Tabs

## 1. Executive Funnel Overview

Purpose:

Provide a high-level view of the browsing-to-enrollment funnel.

Recommended visuals:

- Funnel chart from program list view to purchase or enrollment
- Conversion rate by funnel stage
- Week-over-week funnel drop-off trend
- Overall trial CTA click rate
- Overall evaluation CTA click rate

Key questions:

- Where is the largest drop-off?
- Is the funnel improving after UX changes?
- Are more users taking low-risk trial actions?

## 2. Course Browsing Behavior

Purpose:

Measure whether users can evaluate courses before sign-up.

Recommended visuals:

- Course detail view volume by grade level and program type
- Course detail to sign-up rate
- Pricing view rate before sign-in
- Course selection rate by program type
- Top pages by exit rate

Key questions:

- Which course pages generate the most interest?
- Are users seeing pricing before they register?
- Which program types create the most browsing friction?

## 3. Evaluation and Trial Paths

Purpose:

Compare high-commitment evaluation behavior with low-risk trial behavior.

Recommended visuals:

- Evaluation CTA clicks by page type
- Trial CTA clicks by page type
- Evaluation start to completion rate
- Trial CTA click to purchase or enrollment rate
- Segment comparison: users exposed to trial CTA vs not exposed

Key questions:

- Does surfacing the trial option reduce evaluation-stage drop-off?
- Are users choosing trial before evaluation?
- Which pages are best for placing trial CTAs?

## 4. Form and Sign-Up Performance

Purpose:

Monitor registration and form completion quality.

Recommended visuals:

- Sign-up start to form submit rate
- Form start and submit counts by form name
- Form abandonment by first field
- Sign-up conversion by entry point
- Error or incomplete submission rate if available

Key questions:

- Which forms create the most friction?
- Are form identifiers complete enough for reliable reporting?
- Which entry points drive the best registration completion?

## 5. Purchase and Policy Confidence

Purpose:

Measure whether risk-reducing information supports purchase confidence.

Recommended visuals:

- Checkout start rate
- Purchase or enrollment conversion rate
- Refund/drop policy view rate
- Policy view to checkout rate
- Conversion comparison for users who viewed policies vs users who did not

Key questions:

- Do users who view refund or drop policies convert differently?
- Are policy pages supporting confidence or exposing late-stage concern?
- Does better policy visibility reduce purchase-stage drop-off?

## Core KPI Definitions

| KPI | Definition |
| --- | --- |
| Course detail to sign-up rate | Sign-up starts divided by course detail views |
| Pricing visibility rate | Course detail views with pricing visible divided by total course detail views |
| Trial CTA click rate | Trial CTA clicks divided by eligible page views |
| Evaluation CTA click rate | Evaluation CTA clicks divided by eligible page views |
| Evaluation completion rate | Evaluation completions divided by evaluation starts |
| Form completion rate | Form submits divided by form starts |
| Checkout start rate | Checkout starts divided by course detail views or sign-up completions |
| Enrollment conversion rate | Purchases or enrollments divided by course detail views, sign-up starts, or checkout starts depending on funnel level |
| Policy view rate | Refund/drop policy views divided by eligible purchase-intent sessions |

## Recommended Segments

- Entry source
- New vs returning users
- Grade level
- Program type
- Users exposed to trial CTA vs not exposed
- Users who viewed pricing before sign-in vs users who did not
- Users who viewed refund/drop policy vs users who did not
- Users who started evaluation vs users who skipped or abandoned evaluation

## Post-Launch Experiment Ideas

- Test adding trial CTA modules to course pages
- Test showing price ranges before sign-in
- Test supportive evaluation language against pass/fail-oriented language
- Test adding refund/drop policy snippets near checkout
- Test moving evaluation prompts later in the browsing flow

## Reporting Cadence

- Weekly: monitor funnel conversion, trial clicks, evaluation starts, and checkout starts
- Biweekly: review segment-level changes and page-level friction
- Monthly: evaluate whether UX changes improved conversion and whether additional experiments are needed
